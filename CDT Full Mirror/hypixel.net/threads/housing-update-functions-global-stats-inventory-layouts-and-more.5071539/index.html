Housing Update - Functions, Global Stats, Inventory Layouts, and more!

  • Thread starter ConnorLinfoot
  • Start date Sep 12, 2022

  • Sep 12, 2022
  • Replies: 158


Hey!

Today we've released a new Housing update that includes many community-requested features and changes! This update includes:
  • Functions
  • Global Stats
  • Inventory Layouts
  • Game Rules
  • New Event Actions
  • and more!
Continue reading to learn more!

► Functions
Functions are a new action system that can be called from any other action system. They can also be used for multiple functions, making reusing complex action setups even easier to use in multiple places. You can manage your functions in the "House Settings" menu.



► Global Stats
Global Stats are a much-requested feature and we've included it in this update for that reason. Like the existing player stats system, these stats can be modified by actions, checked via conditions, and managed by commands. Global stats do not expire and are stored with the house permanently and are only removed if set to zero.


► Inventory Layouts
As most players are probably aware, the current methods for providing an inventory layout to players aren't great. The only real option is the PVP layout, which is heavily limited in only supporting vanilla items and only being applied in PVP areas. With this update, we're adding a new system, "Inventory Layouts." This system allows the creation of multiple layouts with support for custom items that can be applied at any time with actions.


For now, the existing PVP layout system still exists; in the future, we will likely look at removing this, as the same functionality can be replicated with the new system and event actions.

► Game Rules
Game Rules is a new settings system that allows toggling more small options, which can be toggled via the command /gamerules. The new options available with this command are:
  • despawnVehicleDismount
    • Whether vehicles should despawn when a player dismounts them.
  • itemDrops
    • Whether items should drop in the world, either from players or breaking blocks.
  • playerCrafting
    • Whether players can use crafting menus.
  • eggHuntAllowFlying
    • Whether flying is allowed when doing an Egg Hunt.
More settings may be added going forward based on ideas and suggestions!

► New Event Actions
We've also added some requested event actions, as well as some others we think can be useful!
  • Player Quit
    • Executes when a player leaves the house.
    • This event is very limited with actions due to the player no longer being on the instance. The primary intended use for this is for modifying stats.
    • This event does not trigger during a reinstance.
  • Nether Portal Use
    • Executes when a player enters a nether portal, similar to entering a nether portal, there is a short delay for players in survival.
  • Player Damage
    • Executes when a player takes damage by any means.
    • Can be canceled using the "Cancel Event" action.
    • Includes a Damage Cause condition to check the type of damage that occurred.
  • Player Block Break
    • Executes when a player attempts to break a block in the plot. (This event does not run if a player successfully breaks a block due to build permissions)
    • Includes a Block Type condition to check which type of block has been broken.
Some of these new event actions are more complex and can trigger more often than previous events; we may tweak them or add cooldowns if needed and will be monitoring usage.

► Offline Building
With this update, we have also introduced the ability to enable offline building with a new permission that can be given to groups. This is a major change for Housing, especially in regards to moderation. We'll be monitoring the impact of this change and may make further changes if necessary.

Along with this, some systems are now accessible for non-owners with build permissions:
  • /save can now be ran by anyone with build permission
  • NPCs can now be modified
  • Conditionals can now be modified
  • Players with build can now access more of the items in the items menu.

► Other Changes
  • Some text input systems have been moved to use the player chat, allowing longer messages for these systems.
  • Owners now bypass max players of their house and can still join even when full.
  • Added the /menu command as a shortcut to open the Housing Menu at any time, even without the item.
  • Added the /limits command to view limits and their values for different systems.
  • Added the "ping" placeholder.
  • NPCs can now have equipment equipped.
  • Added new pro tools, //walls, and //wireframe.
  • Pro Tools selection will now show with particles, this setting can be changed in the new Housing tab of /settings (when in a house)
  • Added the ability to toggle flags on items in /edit.
  • Player States and Persistent Stats are now kept for 14 days.
  • Increased limits for player stats to 40.
  • When a stat changes to zero, it will no longer count towards the stats limit.
Action and Condition Changes:
  • Added an "Exit" action, available in conditionals and random actions.
  • Added a PVP-enabled condition for the PVP State Change event action.
  • Added an option to conditional actions to match any condition rather than requiring all to be matched.
  • Added the ability to increment and decrement health with the Change Health action (previously known as Set Health).
That's it for this update, as always keep suggestions coming and report bugs here!
